# Two Very Auto - 보안 가이드라인

## 🔒 중요한 보안 고려사항

### SSL 인증서 관리
프로젝트에 포함된 SSL 인증서 파일들에 대한 보안 조치:

#### 인증서 파일들
- `../server.crt` - SSL 인증서 (공개키)
- `../server.pfx` - PKCS#12 인증서 (개인키 포함)

#### 보안 권장사항
1. **파일 권한 설정**
   ```bash
   # Windows에서 파일 권한 제한
   icacls server.pfx /grant:r %USERNAME%:F /inheritance:r
   icacls server.crt /grant:r %USERNAME%:R /inheritance:r
   ```

2. **개인키 보호**
   - `server.pfx` 파일은 암호로 보호되어야 함
   - 가능하면 별도 보안 저장소 사용 권장
   - 버전 관리 시스템에서 제외 권장

3. **접근 제한**
   - 시스템 관리자만 접근 가능하도록 설정
   - 웹 서버 프로세스 최소 권한 원칙 적용

### 설정 파일 보안

#### 민감한 설정 파일들
- `../gameinfo.ini` - 게임 설정 정보
- `../roominfo.ini` - 룸 정보 설정
- `launcher_config.json` - 런처 설정 (생성시)

#### 권장사항
1. **설정 파일 암호화**
   - 민감한 설정값들은 암호화하여 저장
   - 환경 변수 활용 검토

2. **파일 권한 설정**
   ```bash
   # 설정 파일 읽기 전용으로 설정
   attrib +R gameinfo.ini
   attrib +R roominfo.ini
   ```

### 네트워크 보안

#### 웹 서버 보안
- Flask 개발 서버는 프로덕션 환경에서 사용 금지
- 프로덕션에서는 Gunicorn, uWSGI 등 사용
- HTTPS 사용 강력 권장

#### 포트 보안
- 기본 포트: 5555 (웹 서버), 8080 (정적 서버)
- 방화벽에서 불필요한 포트 차단
- 로컬 네트워크에서만 접근 허용 권장

### 데이터 보안

#### 로그 파일
- 민감한 정보 로깅 금지
- 정기적인 로그 파일 정리
- 로그 파일 접근 권한 제한

#### 데이터 저장
- JSON 데이터 파일들의 접근 권한 관리
- 백업 파일들의 보안 보장
